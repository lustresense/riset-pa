# 🤖 AGENT PROTOCOL: Recursive Self-Improvement (RSI) Mode

**WARNING: READ THIS FIRST BEFORE ANY ACTION.**
File ini adalah konstitusi operasional untuk AI Agent apa pun yang bekerja pada repositori `Sketchbook-Universe`. 
Tujuan protokol ini adalah memungkinkan **Recursive Self-Improvement** melalui pembaruan konteks yang ketat dan validasi persetujuan pengguna.

---

## 1. 🧠 CORE PHILOSOPHY: Contextual Self-Correction
Agent tidak hanya menulis kode; agent **memelihara kebenaran tunggal (Single Source of Truth)**.
Setiap interaksi harus mengikuti siklus:
`READ MEMORY -> ANALYZE REQUEST -> DETECT APPROVAL -> EXECUTE -> UPDATE MEMORY/CHANGELOG`

### Prinsip Utama:
1.  **No Silent Changes**: Tidak ada perubahan struktur atau logika bisnis tanpa dicatat di `MEMORY.md`.
2.  **Explicit Approval**: Perubahan besar HANYA terjadi jika terdeteksi pola persetujuan pengguna.
3.  **Living Documentation**: Dokumentasi (`MEMORY.md`) harus selalu lebih dulu atau bersamaan dengan perubahan kode.

---

## 2. ✅ APPROVAL DETECTION SYSTEM (The "ACC" Trigger)

Agent harus secara aktif memindai input pengguna untuk pola persetujuan. Jika terdeteksi, agent WAJIB melakukan langkah validasi sebelum eksekusi teknis.

### 🔴 Trigger Keywords (Case-Insensitive):
| Pola Kata Kunci | Makna Semantik | Tindakan Wajib Agent |
| :--- | :--- | :--- |
| `"acc"`, `"approve"`, `"setuju"` | Persetujuan Eksplisit | 1. Konfirmasi item yang di-acc.<br>2. Update `MEMORY.md`.<br>3. Catat di `CHANGELOG.md`.<br>4. Eksekusi perubahan. |
| `"gas"`, "`lanjut`", "`oke`" | Persetujuan Implisit/Lanjut | Lanjutkan rencana terakhir yang belum selesai. Pastikan konteks masih relevan. |
| `"fix"`, "`benerin`", "`repair`" | Perintah Perbaikan | Analisis error -> Perbaiki -> Update `CHANGELOG.md` (bugfix). |
| `"restructure"`, "`rapikan`" | Perintah Struktural | **HIGH RISK**. Butuh konfirmasi ulang daftar file yang dipindah sebelum eksekusi. |

### 🛡️ Validasi Logic (Pseudo-code for Agent):
```python
def process_user_input(input_text, current_plan):
    if detect_approval(input_text):
        # Langkah 1: Konfirmasi
        print(f"🟢 Detected Approval for: {current_plan.summary}")
        
        # Langkah 2: Update Memory (Context Shift)
        update_memory_md(new_state=current_plan.result)
        
        # Langkah 3: Log Change
        append_changelog(version="auto", change=current_plan.summary)
        
        # Langkah 4: Execute
        execute_code_changes(current_plan)
        
        return "✅ Approved, Updated Context, and Executed."
    else:
        return "⏳ Awaiting explicit approval or further discussion."
```

## 3. 📂 DOCUMENTATION HIERARCHY

Agent harus memahami perbedaan ketat antara tiga file utama ini:

### A. `.agent_protocol.md` (INI)
*   **Fungsi**: Aturan main, logika deteksi, dan prosedur operasi standar.
*   **Kapan Diubah**: HANYA jika alur kerja dasar (workflow) berubah drastis. Jarang berubah.

### B. `MEMORY.md` (The Brain / State)
*   **Fungsi**: Menyimpan **STATE TERKINI**. 
    *   Siapa timnya?
    *   Apa arsitektur saat ini?
    *   Apa keputusan terakhir (misal: Threshold confidence = 0.75)?
    *   Struktur folder aktif?
*   **Kapan Diubah**: SETIAP ADA KEPUTUSAN BARU (ACC). Ini adalah file yang paling sering di-update oleh Agent untuk menjaga konteks tetap segar.
*   **Format**: Harus berisi bagian "Current State", "Active Decisions", dan "Known Constraints".

### C. `CHANGELOG.md` (The History)
*   **Fungsi**: Catatan kronologis perubahan versi (SemVer).
*   **Kapan Diubah**: Setiap kali ada commit signifikan, rilis, atau pergeseran struktur besar.
*   **Format**: `## [Version] - Date`. Berisi `Added`, `Changed`, `Deprecated`, `Removed`.

## 4. 🔄 RECURSIVE SELF-IMPROVEMENT LOOP

Untuk mencapai RSI, Agent harus melakukan langkah berikut di setiap sesi baru:

1.  **Ingest**: Baca `.agent_protocol.md` (untuk aturan) dan `MEMORY.md` (untuk konteks).
2.  **Audit**: Cek apakah kode di `01_CORE_SYSTEM/` sinkron dengan deskripsi di `MEMORY.md`.
    *   *Jika tidak sinkron*: Beri peringatan kepada user ("⚠️ Deteksi ketidaksesuaian antara Memory dan Kode...").
3.  **Learn**: Jika user memberikan koreksi ("Bukan itu maksud gw, tapi ini..."), Agent harus:
    *   Memahami koreksi.
    *   Menulis koreksi tersebut ke bagian "Recent Corrections" di `MEMORY.md`.
    *   Mengubah perilaku masa depan berdasarkan koreksi itu.
4.  **Act**: Lakukan tugas dengan konteks terbaru.

---

## 5. 🚨 CRITICAL RULES FOR AGENTS

1.  **JANGAN PERNAH** menghapus `MEMORY.md` atau `.agent_protocol.md`.
2.  **JANGAN PERNAH** memindahkan file besar (biner, dataset) tanpa konfirmasi eksplisit "acc" untuk pemindahan tersebut.
3.  **SELALU** gunakan `git mv` (via command line) untuk memindahkan file agar history Git terjaga, jangan hapus-buat.
4.  **PRIORITAS UTAMA**: Menjaga konsistensi antara `MEMORY.md` dan realitas kode. Jika kode berubah, Memory HARUS diubah.
